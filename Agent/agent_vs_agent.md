# 推荐系统里的多Agent强化学习 (MARL)

## 什么是 MARL？

MARL（Multi-Agent Reinforcement Learning，多智能体强化学习）是指在推荐系统中引入多个智能体，每个智能体可以代表不同的角色（如用户、物品、平台等），通过强化学习来优化推荐策略。

---

## MARL 与 Agent 自动调参的异同点

### 相同点

- **基于强化学习框架**：两者都使用强化学习的思想（状态、动作、奖励）来优化决策
- **目标导向**：都旨在提升系统性能，如点击率、用户满意度等
- **探索与利用**：都需要在探索新策略和利用已知策略之间取得平衡

### 不同点

| 维度 | MARL | Agent 自动调参 |
|------|------|----------------|
| **核心目标** | 多智能体协作/博弈，优化整体推荐效果 | 单/多Agent调整系统超参数 |
| **应用场景** | 推荐排序、竞价广告、用户建模 | 神经网络结构搜索、激活函数选择 |
| **交互对象** | 用户、物品、其他Agent | 训练流水线、模型配置 |
| **复杂度** | 高（多智能体博弈、通信） | 中等（搜索空间遍历） |

---

## 总结

MARL 更侧重于**多方交互**场景，而 Agent 自动调参更侧重于**系统优化**。两者都是当前推荐系统领域的热门研究方向。
