# 训练计划

## 第 1 天：重新理解 RL 的最小结构

### 目标
彻底清楚 RL 到底在解决什么问题。

### 任务（30 分钟）

拿一张纸写下 RL 的最小结构：

- **state**  $` s `$
- **action** $`a`$
- **reward** $`r`$
- **transition** $`P(s' \mid s, a)`$
- **policy** $`\pi(a \mid s)`$

然后做一个练习：  
把下面三个问题写成 RL：
1. 推荐排序  
2. 自动驾驶  
3. 下围棋  

每个问题都写：
- state =
- action =
- reward =
- transition =
- horizon =
- 关键问题

**问自己**：action 是否真的影响未来 state？  
这是 RL 是否必要的核心。

### 练习
#### 推荐系统
state = 用户画像信息，包括用户的基本信息和行为历史列表，压缩成embedding  
action = 推荐的item list，请求级
reward = request的点击/观看时长
transition = 下一个用户状态，包含了上一次点击的item  
horizon = request  
  
request就是用户每刷一次，作为一条(s,a,r)来处理。  
  
  
state = 用户画像信息，包括用户的基本信息和行为历史列表，压缩成embedding  
action = 推荐的item list, session级
reward = session级点击/观看时长
transition = 下一个用户状态，包含了上一个session点击的item  
horizon = session  
  
session就是用户打开app，刷多次，最后关闭。a是整个session的推荐结果。

### 环境导致指什么？
首先，机器人九宫格问题里，什么是环境？什么是agent？  
agent=机器人  
环境=?  
环境这个定义有两层，比较混淆，一层是：状态空间S(包括所有可能状态)、P(r|s,a)、P(s'|s,a)，也就是规则，这层比较抽象  
另一层是：机器人当前的状态(位置)、p(r|s,a)、p(s'|s,a),也就是规则下的一个切片、一个状态。
说：机器人执行动作后"环境变化"，指的是后者变了，不是环境的规则变了。  
  
agent能控制的只有一个，就是策略pi(a|s)  
状态是环境的一部分，而不是agent的一部分。状态说的是环境当前的状态，不是agent的状态。  
状态state是对过去历史的压缩。马尔科夫要求，未来的行为和状态只取决于当前状态，和之前的状态完全无关。所以，当前状态必须能完全代表过去历史上的所有状态。对推荐系统来说，就是要把过去用户的行为全部压缩进状态。对九宫格来说，机器人的历史把机器人带到了当前状态，当前状态也就完全能代表过去的所有行为。
  
推荐系统里，什么是环境？什么是agent？
推荐系统本身是agent，它负责做决策，决策就是当前状态下推荐哪些item，决策的空间是候选item。
环境=用户当前的状态(一种说不清道不明的东西，不像机器人问题有明确的代指)、p(r|s,a)：给现在的用户推了一些item，获得用户奖励的概率、p(s'|s,a)：用户当前状态给用户推了几个item用户跳转到下一状态的概率。  
  
理解了，推荐系统里用rl的问题。你推什么，对用户的状态有影响吗？用户会因为看了几个视频就受到影响了吗？部分情况成立。比如用户没看过某个电影，你给他推了这个电影，用户状态会改变，会特别想看这个电影相关的内容。  
  
### bandit和rl什么区别？
核心点是：agent执行action之后，环境有没有变化。  
上面要确定好环境是什么，就是为了回答这个问题。  
对bandit来说，拉动哪个杆，环境：也就是老虎机返回奖励的概率

## 第 2 天：真正理解 Bellman Equation

很多人会背公式，但没有直觉。

**核心公式：**

$$`V(s) = \mathbb{E}\bigl[r + \gamma V(s')\bigr]`$$

### 任务（30 分钟）

自己构造一个简单环境：

- **状态**: $`s_0 \to s_1 \to s_2 \to`$ terminal  
- **reward**: 0, 0, 10  

手算：

- $`V(s_2)`$
- $`V(s_1)`$
- $`V(s_0)`$

然后再想：如果 $`V(s_2)`$ 估计错了会怎样？

你会直观看到：**误差向前传播**，这就是 TD-learning 不稳定的根源。

## 第 4 天：推荐系统 RL 拆解

今天专门拆推荐问题。

### 任务

写出推荐系统的三个版本：

1. **Supervised Learning**  
   $`(user, item) \to`$ click probability  

2. **Contextual Bandit**  
   context $`\to`$ choose item $`\to`$ observe click  

3. **RL**  
   state $`\to`$ item $`\to`$ user response $`\to`$ next state  

然后问自己：**action 是否改变未来状态？**

这一步会让你理解为什么推荐 RL 很难。

## 第 5 天：重新实现一个最小 RL

不用深度网络。

### 任务（30 分钟）

写一个 20 行 Q-learning：

**环境：**

- 3 个状态  
- 2 个动作  

**打印：** $`Q(s,a)`$ 每一步变化  

**观察：** Q value 如何传播  

这会让你对 Q-learning 的理解非常直观。

## 第 6 天：论文拆解训练

选一篇 RL 论文（推荐系统或 LLM agent 都行），然后只回答 5 个问题：

1. state 是什么  
2. action 是什么  
3. reward 是什么  
4. transition 是否受 action 影响  
5. 为什么不用 bandit  

如果你回答不出来，说明论文在模糊建模。  
这个训练非常有效。

## 第 7 天：统一视角

今天的任务是建立一个统一框架。

画一条线：

```
Supervised Learning
        ↓
Contextual Bandit
        ↓
Offline RL
        ↓
Online RL
```

然后把这些问题放进去：

- 推荐排序  
- 广告系统  
- 自动驾驶  
- 机器人  
- 游戏 AI  
- LLM Agent  

你会看到：不同问题在光谱的不同位置。

---

## 这个计划完成后的效果

7 天之后你会发生一个明显变化。

**以前**看到 RL 论文：看算法  

**之后**会变成：先看问题建模  

也就是：

- state 是否完整  
- action 是否影响未来  
- reward 是否延迟  

一旦这三点清楚，该用 RL 还是 Bandit 会变得很直觉。

## 额外：高手习惯

这个思维习惯非常关键。
