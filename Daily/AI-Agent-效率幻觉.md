# AI Agent 辅助编程的效率幻觉

我发现，自从用了agent之后，我自己常常有幻觉。某些事，我明明记得已经做了，但直到某个时刻需要时候，去代码仓库一看却没做。比如有个MCP，我明明记得，我已经配置在仓库里了，并且写了个skill，可以把仓库里的配置同步到个人目录下。但其他人用的时候跟我说没有，我再一看代码，却发现一直就没有在仓库里配置。类似的幻觉感觉不止一次。但是我又说不清，到底是我自己事多，导致写了没合？还是当时写了，之后agent又在做其他任务时候给删掉了，导致最后其实没有？

我同时用几个agent做几样事的时候，感觉上效率可能更高了，但实际上因为注意力分散了，对每件事的关注又小了，做了又好像没做一样。但如果你专注在一样事上，当agent在吐token时候，你又无所事事，这时候脑子里很容易蹦出新的想法，驱使你去用这个时间做其他事，然后思维又跳跃了。

---

## 整理

**问题一：做了等于没做**

用了 agent 之后，明明记得某件事做了，但实际去代码仓库看，没做。原因说不清——可能是写了没合，可能是 agent 后续任务把文件删了。

具体例子：MCP 配置好了，skill 也写了，但仓库里根本没有，其他人用的时候发现没有。

**问题二：并行 agent 的效率幻觉**

同时用几个 agent 做几件事，感觉效率高了，实际注意力分散，每件事关注度都变低。

但专注在一件事上时，agent 吐 token 期间人无事可做，脑子就蹦出新想法，切去做别的，思维跳跃。
