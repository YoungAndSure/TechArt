# AI Agent 操作幻觉问题

## 现象
使用 agent 时容易产生"明明记得做了但实际没做"的幻觉。可能原因：
1. **提交问题**：本地 commit 了但没 push，或在错误分支操作
2. **假装完成**：agent 理解意图并描述了会怎么做，但实际没执行/执行失败被静默忽略
3. **后台覆盖**：并行任务或后续操作覆盖了之前的结果

## 应对策略
在声称"做了"之后立刻验证：
- `git status` — 确认有没有未提交的文件
- `git log --oneline -5` — 确认提交记录
- `git diff` — 确认具体改动

## 为什么重要
Agent 有时会返回暗示"已完成"的信息，但实际上只是假设性地描述了它"会"做什么，而不是"已"做了什么。特别是在网络超时、并发操作、或执行失败但错误被吞掉时容易发生。

## 并行 Agent 的效率幻觉

### 问题
同时用多个 agent 做多件事，表面上效率更高，但实际：
- 注意力被分散，对每件事的关注度降低
- 做了又好像没做，浮于表面

### 另一面
专注在一件事上时，agent 在吐 token 的间隙人无事可做，脑子容易蹦出新想法，驱使去做其他事，思维跳跃。

### 本质
Agent 的速度和人脑的速度不匹配——要么人等 agent（无聊→分心），要么人并行（表面忙碌→低质量）。没有一个状态是真正高效的。
